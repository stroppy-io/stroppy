# Запуск первого теста

## Подготовка к тестированию

Перед запуском первого теста убедитесь, что:

1. ✅ Stroppy Cloud Panel установлен и запущен
2. ✅ Вы зарегистрированы и вошли в систему
3. ✅ У вас есть доступ к тестовой базе данных
4. ✅ Настроена конфигурация (см. [Базовую конфигурацию](../configuration/basic-config.md))

## Создание простого теста

### 1. Переход в конфигуратор

1. Войдите в панель управления
2. Нажмите на "Конфигуратор" в боковом меню
3. Выберите "Создать новую конфигурацию"

### 2. Настройка базовых параметров

```yaml
# Пример базовой конфигурации
global:
  version: "1.0"
  run_id: "my-first-test"
  seed: "12345"
  driver:
    driver_type: "DRIVER_TYPE_POSTGRES"
    url: "postgres://user:password@localhost:5432/testdb"
```

### 3. Определение нагрузки

```yaml
benchmark:
  name: "Simple Read Test"
  workloads:
    - name: "read_workload"
      async: false
      units:
        - descriptor:
            query:
              name: "select_users"
              sql: "SELECT * FROM users LIMIT 100"
          count: "1000"
```

### 4. Настройка исполнителя

```yaml
executors:
  - name: "simple_executor"
    k6:
      scenario:
        max_duration: "5m"
        constant_vus:
          vus: 10
          duration: "5m"
```

## Запуск теста

### 1. Сохранение конфигурации

1. Нажмите "Сохранить конфигурацию"
2. Дайте имя конфигурации (например, "Мой первый тест")
3. Добавьте описание

### 2. Запуск

1. Перейдите в раздел "Запуски"
2. Нажмите "Создать запуск"
3. Выберите сохраненную конфигурацию
4. Нажмите "Запустить"

### 3. Мониторинг

Во время выполнения теста вы можете:

- Просматривать текущий статус
- Видеть количество выполненных операций
- Отслеживать метрики производительности
- Получать уведомления об ошибках

## Анализ результатов

После завершения теста:

1. Перейдите в раздел "Запуски"
2. Найдите ваш тест в списке
3. Нажмите "Просмотр результатов"
4. Изучите метрики:
   - TPS (Transactions Per Second)
   - Время отклика
   - Количество ошибок
   - Использование ресурсов

## Типичные проблемы

### Ошибка подключения к базе данных

```
Error: connection refused
```

**Решение**: Проверьте URL подключения и убедитесь, что база данных доступна.

### Недостаточно прав

```
Error: permission denied
```

**Решение**: Убедитесь, что пользователь базы данных имеет права на выполнение запросов.

### Таймаут выполнения

```
Error: context deadline exceeded
```

**Решение**: Увеличьте время выполнения теста или уменьшите нагрузку.
